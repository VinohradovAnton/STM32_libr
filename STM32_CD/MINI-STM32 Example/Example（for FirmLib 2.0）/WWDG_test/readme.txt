1 设计要求
	与独立看门狗应用实例类似。利用窗口看门狗的提前唤醒中断服务子程序来进行独立看门狗计数值的
	重载，每次重载一次计数值LED2就闪烁一次。通过按键KEY来阻止计数值的重载，导致独立看门狗的复
	位，用LED1灯亮表示系统从窗口看门狗复位中恢复。

2 硬件电路
	硬件电路采用与7.1小节应用实例一样硬件电路。其中Key按钮用于通过PB9产生一个外部中断，LED1、
	LED2则用于显示窗口看门狗状态。

3 软件程序设计
	根据设计任务要求，程序内容主要包括：
	(1)	配置并启动窗口看门狗，配置PB口第9个引脚作为外部中断；
	(2)	系统启动时检测系统是否从WWDG复位恢复，如果曾经发生过则点亮LED1；
	(3)	在窗口看门狗的提前唤醒中断的中断服务子程序中喂狗，并闪烁LED2；
	(4)	在EXTI9中断（KEY按钮触发）的中断返回时不清除中断挂起位，导致窗口看门狗的提前唤醒中断
	无法进入，最终导致因无法重载看门狗计数器而系统复位；
	整个工程包含3个源文件：STM32F10x.s、stm32f10x_it.c和main.c，其中STM32F10x.s为启动代码，
	所有中断服务子程序均在stm32f10x_it.c中，其它函数则在main.c中。下面分别介绍相关的函数，
	具体程序清单见参考程序。
函数WWDG_IRQHandler是提前唤醒中断的中断服务程序，用于定时重载窗口看门狗计数器，并闪烁LED2。
函数XTI9_5_IRQHandler是按下KEY按键所触发的中断服务程序，在返回时不清除中断挂起位。
其他一些函数，例如SysTick_Configuration、NVIC_Configuration等，这里就不再冗述了。

4 运行过程
	(1)	使用Keil uVision3 通过ULINK 2仿真器连接实验板，打开实验例程目录WWDG_TEST子目录下的
	WWDG.Uv2例程，编译链接工程；
	(2)	将程序下载到Flash中，然后按评估板上Reset按键，可以看到LED1不亮，LED2不断闪烁；
	(3)	按下KEY按键，可以看到经过非常短暂时间之后，LED1被点亮，LED2不断闪烁。
